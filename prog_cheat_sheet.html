Generators: Both Python and JavaScript support generators for lazy evaluation and memory efficiency. Python offers generator expressions (x for x in iterable) for inline generator creation, while JavaScript requires generator functions function*. Generators are ideal for processing large datasets, infinite sequences, or streaming data without loading everything into memory. Use yield (Python) or yield with function* (JavaScript) to pause execution and produce values one at a time.

Generator State Preservation: The generator object stores all local variables and the execution position between yield statements. When you call next(), it resumes from the last yield with all variables intact. JS Integration Patterns with widget-tweaks:

1. Wire data-* attributes for vanilla JS / Alpine.js:
{% render_field form.qty type="number" min="1" data-stock="{{ product.stock }}" x-model="qty" %}
Alpine's x-model binds reactively; data-stock is read by JS for client-side cap validation.

2. HTMX — partial updates without a full page reload:
{% render_field form.search hx-get="{% url 'search' %}" hx-trigger="keyup changed delay:300ms" hx-target="#results" hx-swap="innerHTML" class="form-control" %}
Typing in the field triggers an AJAX GET; the response HTML replaces #results. No JS event listeners needed.

3. Dynamic id / name for formsets:
{% render_field form.title id="id_form-{{ forloop.counter0 }}-title" name="form-{{ forloop.counter0 }}-title" %}
Inline formsets need unique ids/names; render_field lets you interpolate loop variables directly in the attribute value.

4. Conditional readonly driven by context:
{% if is_readonly %}{{ form.email|attr:"readonly:true"|add_class:"bg-light" }}{% else %}{{ form.email|add_class:"form-control" }}{% endif %}

5. Combining multiple filters:
{{ form.phone|add_class:"form-control"|attr:"type:tel"|attr:"data-mask:phone"|add_error_class:"is-invalid" }}
Filters are applied left-to-right; the final rendered <input> carries all accumulated attributes.

  • Works with CSP script-src policies (no inline JS execution)

CSS Combinators

Name Syntax Selects Example Notes
🔗 BASIC COMBINATORS
Descendant A B All B elements inside A (any depth) div p Most common; applies to deeply nested children too
Child A > B Direct children B of A (one level only) ul > li Stricter than descendant; skips grandchildren
Adjacent Sibling A + B The first B immediately after A (same parent) h1 + p Only matches the very next sibling
General Sibling A ~ B All B elements after A (same parent) h1 ~ p Matches all subsequent siblings, not just the first
📋 COLUMN COMBINATOR (Experimental)
Column A || B B elements that belong to the column defined by A col.selected || td Experimental; targets cells within a <col> or <colgroup>
⚡ QUICK COMPARISON
div p Descendant Selects all <p> anywhere inside <div>
div > p Child Selects only <p> that are direct children of <div>
div + p Adjacent Sibling Selects the first <p> immediately after <div>
div ~ p General Sibling Selects all <p> after <div> at the same level
Tip: Combinators can be chained, e.g. nav > ul > li a selects all <a> inside <li> that are direct children of a <ul> direct child of <nav>. Prefer the child combinator (>) over the descendant combinator for better performance and more predictable specificity.

CSS Variables (Custom Properties)

Concept Syntax Description Example
📌 DEFINING & USING VARIABLES
Define a variable --name: value; Declare inside a selector (usually :root for global scope) :root { --primary: #3490dc; }
Use a variable var(--name) Reference the variable value anywhere a value is accepted color: var(--primary);
Fallback value var(--name, fallback) Second argument is used if the variable is not set color: var(--accent, rebeccapurple);
Chained fallback var(--a, var(--b, default)) Nest var() calls for multiple fallbacks color: var(--theme, var(--primary, black));
🔁 SCOPE & INHERITANCE
Global scope :root { --x: value; } Available everywhere in the document :root { --spacing: 8px; }
Local scope .card { --x: value; } Only available inside .card and its descendants .card { --bg: white; background: var(--bg); }
Override / override in theme Redefine in a narrower selector Child selector redefines the variable; cascades down from there .dark { --primary: #90cdf4; }
Inheritance Automatic Variables inherit naturally like other CSS properties Child elements see parent's --x unless overridden
⚙️ JS INTEROP
Read variable in JS getComputedStyle(el).getPropertyValue('--name') Returns the computed value as a string getComputedStyle(document.documentElement).getPropertyValue('--primary')
Set variable in JS el.style.setProperty('--name', value) Sets an inline variable on any element document.documentElement.style.setProperty('--primary', '#ff0')
Tip: Custom property names are case-sensitive (--Color ≠ --color). They do not support !important on the declaration itself, but the var() usage can be overridden normally via specificity. Invalid or missing variables render the property as its initial value.

CSS At-Rules (@)

At-Rule Purpose Common Syntax Notes
📥 IMPORTS & FONTS
@import Include an external stylesheet @import url('other.css'); Must be at the top of the file; blocks rendering — prefer <link> in HTML
@font-face Define a custom font @font-face { font-family: 'MyFont'; src: url('my.woff2'); } Use woff2 format for best compression; combine with font-display: swap
📐 RESPONSIVE & CONDITIONAL
@media Apply styles based on viewport / device conditions @media (max-width: 768px) { … } Supports screen, print, prefers-color-scheme, prefers-reduced-motion, etc.
@supports Apply styles only if a CSS feature is supported @supports (display: grid) { … } Use for progressive enhancement; supports not, and, or operators
@container Apply styles based on a parent container's size @container (min-width: 400px) { … } Parent must have container-type set; replaces many @media hacks
@layer Declare cascade layers to control specificity order @layer base, components, utilities; Later layers win over earlier ones regardless of specificity
🎞️ ANIMATIONS & COUNTERS
@keyframes Define animation steps @keyframes slide { from { … } to { … } } Referenced by animation-name; supports percentage steps (e.g. 0%, 50%, 100%)
@counter-style Define custom list counter styles @counter-style thumbs { symbols: '👍'; suffix: ' '; } Used with list-style-type
🔠 TYPOGRAPHY
@charset Declare the character encoding of the stylesheet @charset "UTF-8"; Must be the very first statement; rarely needed for UTF-8 files served with correct headers
@page Modify margins/layout for print pages @page { margin: 1in; } Supports pseudo-classes: :first, :left, :right, :blank
🔮 MODERN / EXPERIMENTAL
@property Register a custom property with type, inheritance and initial value @property --hue { syntax: '<angle>'; inherits: false; initial-value: 0deg; } Enables animating CSS variables; used with @keyframes
@scope Limit styles to a subtree of the DOM @scope (.card) { p { … } } Experimental — avoids naming collisions without BEM
Tip: At-rules that contain blocks (@media, @keyframes, etc.) use { }; statement at-rules (@import, @charset) end with ;. Use @layer to replace specificity wars and keep stylesheets predictable.
