Call Stack Both Python and JavaScript use a call stack:
  • When a function is called, a new frame is pushed onto the stack
  • When a function returns, its frame is popped from the stack
  • Stack overflow occurs when too many nested calls exhaust memory (e.g., infinite recursion)
  • The stack traces error messages from bottom (first call) to top (error location)

Class Decorator def singleton(cls):
    instances = {}
    def get_instance(*args):
        if cls not in instances:
            instances[cls] = cls(*args)
        return instances[cls]
    return get_instance

Key OOP Differences:
  • Python: Rich magic methods (dunder), multiple inheritance, decorators standard, metaclasses, descriptors, @property built-in
  • JavaScript: Prototype-based (class syntax is sugar), single inheritance only, Symbol-based protocols, decorators experimental, no operator overloading
  • Python emphasizes explicit (self), JavaScript emphasizes prototypes and this context
  • Both support modern OOP patterns, but Python offers more metaprogramming capabilities
